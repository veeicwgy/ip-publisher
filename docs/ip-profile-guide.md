# IP Profile Guide

## 为什么要先定义人设

内容自动化的核心不是“让模型更会写”，而是“让模型先知道你是谁”。如果没有稳定的人设输入，系统就只能生成平均风格内容，难以积累个人品牌识别度。

## 字段说明

| 字段 | 含义 | 建议写法 |
| --- | --- | --- |
| `name` | IP 名称/笔名 | 简短、可记忆 |
| `profession` | 职业/领域 | 用一句话说清楚专业身份 |
| `personality` | 性格特征 | 3-5 个形容词即可 |
| `writing_style` | 写作风格 | 描述节奏、语气、证据偏好 |
| `target_audience` | 目标受众 | 明确到职业或圈层 |
| `core_values` | 核心价值观/擅长话题 | 3-6 个高频主题 |
| `taboo_topics` | 禁忌话题 | 不想碰的话题边界 |
| `tone_examples` | 语气样本 | 2-3 句最像你的表达 |
| `preferred_platforms` | 常用平台 | 用平台 ID 填写 |

## 最佳实践

1. `profession` 不要写得过宽，越具体越利于筛热点。
2. `core_values` 最好是长期会重复输出的主题，而不是一次性热点。
3. `tone_examples` 要来自你真实会说的话，不要写成宣传口号。
4. `taboo_topics` 明确后，可以显著减少跑题和风险内容。
