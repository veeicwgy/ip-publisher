# Residue Checklist

## Recommended Search Patterns

在正式修改前和最终交付前，都执行一次全局检索。优先使用下表中的模式，并根据项目词汇补充变体。

| 类别 | 推荐检索词 | 目的 |
| --- | --- | --- |
| 中文收费词 | `升级|付费|收费|定价|套餐|联系升级|查看升级方案` | 找出直接面向用户的旧商业化措辞 |
| 英文收费词 | `pricing|price|upgrade|paid|billing|paywall` | 找出英文 UI 或配置中的商业化残留 |
| 试用锁定词 | `TRIAL_USED|quota|limit|trial` | 找出试用耗尽、次数限制和额度锁定提示 |
| 开源导向词 | `GitHub|仓库|README|开源` | 核对新引导是否已正确接入 |

## Surfaces To Inspect

| 表面 | 重点检查内容 | 期望结果 |
| --- | --- | --- |
| Hero 与首屏 CTA | 主按钮、副按钮、角标、引导短句 | 不再出现价格、升级、销售线索字样 |
| 中段转化区 | 价格卡、方案对比、权益表、联系入口 | 改为仓库、README、快速开始、开源说明 |
| 弹窗与抽屉 | 试用后弹窗、登录后限制弹窗、升级 Dialog | 改为前往 GitHub 仓库或直接移除 |
| 试用状态区 | 试用前提示、试用后提示、已用尽提示 | 语义统一为“查看开源仓库/README” |
| 服务端提示 | `409`、`TRIAL_USED`、额度耗尽响应 | 不再返回升级套餐文案 |
| 测试 | 旧断言、旧常量名、旧 fixture 文案 | 与新导向保持一致 |
| todo 与交付描述 | 待办项、最终总结、检查点描述 | 清晰表达“移除收费导向，改为 GitHub 引导” |

## Acceptable False Positives

如果命中出现在通用 UI 组件源码中，且只是组件名或框架实现细节，例如 `Dialog`、`AlertDialog`、`CommandDialog`，通常不属于业务残留；但如果命中出现在业务页面、后端响应、测试文案或用户可见字符串中，就必须继续处理。

## Completion Criteria

完成清扫前，不要把任务标记为完成。满足下列条件后，才可进入保存检查点和交付阶段。

| 验收项 | 完成标准 |
| --- | --- |
| 业务文案一致性 | 所有用户可见收费/升级文案已移除或改写为 GitHub 引导 |
| 后端响应一致性 | 试用耗尽、额度锁定相关提示与前端导向一致 |
| 自动化验证 | 相关 Vitest 测试通过，且构建成功 |
| 最终交付 | `todo.md` 已勾选完成，并已保存新的检查点 |
