# Multi Publisher

## Trigger when

- 用户说“发布文章”
- 用户说“推送到 XX 平台”
- 用户说“一键发布”
- 用户说“同步到多个平台”
- 用户说“先存草稿再发”

## Goal

根据用户选择的平台，输出可直接复制的标准格式文章、封面与发布前检查结果，帮助用户完成手动分发。

## Supported modes

- 预发布检查：检查标题长度、标签数量、封面尺寸与正文结构是否匹配
- 手动发布包：按平台输出可直接复制粘贴的标题、正文、标签与封面建议

## Supported platforms

- 微信公众号
- 知乎
- 微博
- 今日头条
- CSDN
- 掘金
- 小红书

## 当前限制

- 多平台推送依赖 Wechatsync 浏览器插件，Agent 无法直接调用
- 当前版本：输出可复制的标准格式文章 + 封面，用户手动粘贴发布
- 后续计划：接入平台 API 实现真正自动发布

## Output

| 平台 | 交付内容 | 状态 | 备注 |
| --- | --- | --- | --- |
| 微信公众号 | 标题 + 正文 + 封面建议 | ready | 可直接复制到后台 |

## Rules

- 若用户要求自动发布，先明确说明当前限制，不伪造成功状态。
- 若用户只想生成但不发布，终止在预发布检查。
- 若同一篇文章同步多个平台，保留每个平台的格式差异化版本。
- 若封面或标题不符合平台要求，先给出修正建议，再输出最终可复制版本。
