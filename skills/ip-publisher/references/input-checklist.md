# Input Checklist

生成内容前，必须确认以下输入项。**缺少任意一项都要向用户提问，不能自动填充或使用示例数据。**

## 必填项

| 项目 | 说明 | 缺失时的处理 |
|---|---|---|
| **主题 / 产品名** | 文章要写什么 | 必须向用户提问，无法自动补全 |
| **知识来源 / 内容素材** | 产品文档、官网说明、已有文章等 | 必须向用户提问；未提供时**不**自动抓热点代替，而是说明缺少知识来源无法生成 |
| **主关键词** | 想被哪些词搜索到 | 必须向用户提问；不自动推断 |

## 可选项

| 项目 | 说明 | 缺失时的处理 |
|---|---|---|
| **热点线索** | 想结合的热点或选题方向 | 可省略；省略时围绕知识来源本身生成，不强行蹭热点 |
| **目标读者** | 写给谁看 | 可省略；省略时从内容类型推断 |
| **禁用词** | 不希望出现的词汇 | 可省略 |

## 提问原则

- **一次性合并提问**：把所有缺失项合并成一条消息问清楚，不要分多轮逐一询问。
- **知识库优先**：内容必须来自用户提供的知识来源，不从热点或外部网页凭空生成。
- **审核前置**：内容生成后，必须先完成审核（`audit_report` 状态为 pass），才允许进入发布流程。

## 禁止的自动补全行为

以下行为在默认模式下**不允许**：

- 缺少素材时自动调用 `hotspot-fetcher` 代替知识库
- 缺少关键词时根据主题自动推断并直接使用
- 缺少热点时自动抓取并替用户决策
- 自动写入本地文件（`outputs/`、`request.json` 等）
- 自动执行任何 CLI 命令
