# Output Contract

`ip-publisher` 的默认交付是**对话内结果**，而不是自动执行脚本、自动保存本地文件或直接进入平台后台。最终结果应始终围绕一个清晰、可验证的输出契约组织。

## Required sections

| 区块 | 必须包含的内容 | 目的 |
| --- | --- | --- |
| KB Summary | 产品/工具、知识来源、主关键词、热点线索 | 让用户确认写作输入是否正确 |
| Main Draft | 标题、摘要、Q&A、对比表格、实体标注、审核状态 | 交付 AI 友好主稿 |
| Platform Payloads | 至少一个平台版本的标题、简介、正文、标签建议 | 交付平台可继续编辑的版本 |
| Cover Brief | 封面主题、主标题、副标题、视觉方向 | 让用户后续自行制图或交给设计 |
| Publish Notes | 每个平台的人工检查项、审核结论与发布建议 | 明确这是人工发布准备结果 |

## Prohibited claims

默认结果中不要出现以下说法：

| 不应出现的表述 | 原因 |
| --- | --- |
| “已拿到平台发布回执” | 当前 skill 默认不进入平台后台 |
| “已保存到本地 profile 文件” | 当前 skill 默认不读写本地文件 |
| “已调用仓库脚本生成附件” | 仓库脚本不属于当前已安装 skill 包的默认资产 |
| “已抓取实时热点” | 除非用户明确要求并且结果中标出来源 |
| “已自动发布到多个平台” | 默认只输出发布包或草稿同步说明，不直接执行发布 |

## Companion repo mode

如果用户明确要求进入仓库模式，再额外说明：脚本导出、本地 YAML 保存或外部发布工具接入属于 **companion repo mode**，需要单独确认环境、依赖和人工接管边界。
